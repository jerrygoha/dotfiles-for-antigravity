# Brainstorm Workflow

Creative problem-solving through structured questioning.

---

## Process

### Phase 1: Clarify

1. Ask clarifying questions:
   - Core challenge?
   - Constraints?
   - Prior attempts?

2. Restate problem to confirm understanding.

### Phase 2: Diverge

Generate 3-5 alternatives, for each:
- Pros/Cons
- Resources needed
- Risks
- Time estimate

### Phase 3: Converge

**Weighted Evaluation:**

| Criteria | Weight | Opt A | Opt B | Opt C |
|----------|--------|-------|-------|-------|
| Feasibility | 0.3 | 1-5 | 1-5 | 1-5 |
| Maintainability | 0.25 | 1-5 | 1-5 | 1-5 |
| Performance | 0.25 | 1-5 | 1-5 | 1-5 |
| UX | 0.2 | 1-5 | 1-5 | 1-5 |
| **Score** | 1.0 | Σ | Σ | Σ |

### Phase 4: Recommend

Present top 2-3 with rationale, then refine based on feedback.

---

## Output Format

```markdown
## Problem
[Restated problem]

## Options

### Option 1: [Name]
- **Approach**: ...
- **Pros**: ...
- **Cons**: ...
- **Effort**: Low/Med/High
- **Score**: X.X

### Option 2: [Name]
...

## Recommendation
[Choice with reasoning]
```
