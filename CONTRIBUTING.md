## Getting Started

1. **Fork** the repository
2. **Clone** your fork locally
3. Create a **feature branch** from `main`

## Types of Contributions

### User Rules Templates
- Add new persona templates to `user-rules/examples/`
- Follow existing format with Instructions section
- Include the actual rules in a markdown code block

### Workflows
- Add new workflows to `.agent/workflows/`
- Use YAML frontmatter with `description` field
- Write clear, step-by-step instructions

### Documentation
- Improve existing documentation
- Add translations (create `docs/FILENAME_XX.md`)
- Fix typos and clarify instructions

## Pull Request Process

1. Update documentation if needed
2. Follow commit message conventions:
   ```
   feat: add new user-rules template for DevOps
   fix: correct workflow syntax
   docs: improve Korean translation
   ```
3. Submit PR with clear description

## Code of Conduct

- Be respectful and inclusive
- Welcome newcomers
- Focus on constructive feedback

## Questions?

Open an issue for discussion!
