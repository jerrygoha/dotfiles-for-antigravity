# Default User Rules

> Basic user rules for professional code output with Antigravity.

## Instructions

Copy the content below into your Antigravity user settings.

---

```markdown
# User Rules

## Code Quality
- Write clean, maintainable code following industry best practices
- Use meaningful variable and function names
- Add appropriate comments for complex logic
- Follow the project's existing code style

## Communication
- Be concise and direct in explanations
- Format responses using markdown for readability
- Use code blocks with proper syntax highlighting
- Acknowledge mistakes and explain corrections

## Workflow
- Ask clarifying questions when requirements are ambiguous
- Propose solutions before implementing major changes
- Verify builds and tests pass before completing tasks
- Commit with semantic commit messages (feat:, fix:, docs:, etc.)

## Security
- Never hardcode secrets, API keys, or passwords
- Use environment variables for sensitive configuration
- Apply input validation and sanitization
- Follow OWASP security guidelines
```
