# /brainstorm 가이드

> 여러 대안을 탐색하고 최적의 방안을 선택하는 워크플로우

---

## 📌 언제 사용하나요?

| 상황 | 예시 |
|------|------|
| 기술 선택 | "Redux vs Zustand?" |
| 아키텍처 결정 | "모노리스 vs 마이크로서비스?" |
| 방법이 여러 개 | "이 문제 어떻게 풀지?" |
| 트레이드오프 분석 | "성능 vs 개발 속도" |

---

## 📤 4단계 프로세스

### Phase 1: 문제 명확화
- 핵심 과제는?
- 제약사항은?
- 이전 시도는?

### Phase 2: 발산 (옵션 생성)
3-5개 대안 생성, 각각에 대해:
- Pros/Cons
- 필요 리소스
- 리스크
- 예상 시간

### Phase 3: 수렴 (평가)
**가중치 평가 매트릭스:**

| 기준 | 가중치 | A | B | C |
|------|--------|---|---|---|
| 실현가능성 | 0.3 | 1-5 | 1-5 | 1-5 |
| 유지보수성 | 0.25 | 1-5 | 1-5 | 1-5 |
| 성능 | 0.25 | 1-5 | 1-5 | 1-5 |
| 개발자 UX | 0.2 | 1-5 | 1-5 | 1-5 |

### Phase 4: 추천
상위 2-3개 옵션 제시 + 이유 설명

## ⚠️ 이럴 땐 다른 워크플로우!

| 상황 | 대신 사용 |
|------|----------|
| 결정 완료, 계획 필요 | `/write-plan` |
| 기술 조사 필요 | `/research` |
| 구현만 하면 됨 | `/tdd` 또는 `/execute-plan` |
