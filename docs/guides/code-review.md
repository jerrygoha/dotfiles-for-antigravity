# /code-review 가이드

> 커밋 전 보안 및 품질을 검토하는 워크플로우

---

## 📌 언제 사용하나요?

- 커밋/PR 전 셀프 리뷰
- 보안 취약점 점검
- 코드 품질 체크
- 팀 리뷰 전 사전 점검

---

## 📤 검토 항목

### 🔴 CRITICAL (즉시 수정 - 커밋 금지)
- 하드코딩된 시크릿 (API 키, 비밀번호)
- SQL Injection, XSS 취약점
- 인증/권한 우회 가능성

### 🟠 HIGH (수정 권장)
- 에러 핸들링 누락
- 함수 50줄 초과
- console.log 프로덕션 코드

### 🟡 MEDIUM (개선 권장)
- 불변 패턴 미사용
- TypeScript `any` 사용
- 테스트 누락

---

## ✅ 보안 체크리스트

```
[ ] 하드코딩된 시크릿 없음
[ ] SQL Injection 방어 (파라미터 쿼리)
[ ] XSS 방어 (출력 이스케이프)
[ ] CSRF 보호 활성화
[ ] 모든 입력 유효성 검사
[ ] 에러 메시지에 민감 정보 미포함
```

---

## 🔗 권장 흐름

```
코드 작성 → /code-review → 수정 → /testing → /create-pr
```
