# /pickup 완벽 상세 가이드

> 이전 세션의 컨텍스트를 복원하는 워크플로우

---

## 📌 개요 및 목적

`/pickup`은 `/handoff`로 저장된 **세션 컨텍스트를 복원**하여 이어서 작업하는 워크플로우입니다.

---

## 🎯 언제 사용하나요?

### ✅ 이럴 때 사용하세요

| 상황 | 예시 |
|------|------|
| 새 세션 시작 | "어제 하던 거 이어서" |
| 핸드오프 복원 | `/handoff` 이후 |
| 다른 사람 작업 이어받기 | "누가 하던 거 마저" |

---

## 📤 기대 결과물

```markdown
## 세션 컨텍스트 복원 완료

### 이전 작업 요약
OAuth 기반 사용자 인증 구현 중
- Google OAuth 완료
- GitHub OAuth 진행 중

### Pending Tasks
- [ ] GitHub OAuth 완료
- [ ] 비밀번호 재설정 기능

### 다음 작업
1. GitHub OAuth Provider 설정 완료
2. 비밀번호 재설정 이메일 연동

### 관련 파일
- src/app/api/auth/[...nextauth]/route.ts
- src/lib/prisma.ts

이어서 작업할까요?
```

---

## 📖 상세 사용법

### Step 1: 핸드오프 목록 확인

```bash
ls -la .agent/handoffs/
```

### Step 2: 호출

```
/pickup oauth-auth
```

또는 전체 파일명:

```
/pickup 2026-01-21-oauth-auth
```

### Step 3: 컨텍스트 확인

복원된 내용이 맞는지 확인

### Step 4: 작업 계속

```
"좋아, GitHub OAuth부터 계속하자"
```

---

## ⚠️ 이런 상황엔 다른 워크플로우를!

| 상황 | 대신 사용할 워크플로우 |
|------|----------------------|
| 핸드오프가 없음 | 처음부터 시작 |
| 계획부터 다시 | `/write-plan` |
| 학습된 패턴 확인 | `/learn` 결과물 확인 |

---

## 🔗 함께 사용하면 좋은 워크플로우

```
/pickup              # 컨텍스트 복원
    ↓
작업 계속
    ↓
/handoff             # 다시 저장 (필요시)
```

3. **부분 복원**: 특정 섹션만 참조 가능
