# /refactor-clean 가이드

> 데드 코드를 식별하고 안전하게 제거하는 워크플로우

---

## 📌 언제 사용하나요?

- 기술 부채 정리
- 마이그레이션 후 이전 코드 삭제
- 안 쓰는 의존성 제거
- 프로젝트 전체 정리

---

## 📤 분석 결과 레벨

| 레벨 | 설명 | 조치 |
|------|------|------|
| 🟢 SAFE | 참조 없음, 안전 | 자동 삭제 가능 |
| 🟡 CAUTION | 동적 import 가능성 | 수동 검토 필요 |
| 🔴 DANGER | 설정/진입점 파일 | 삭제 금지 |

---

## 📤 분석 명령어

```bash
# 미사용 exports (TypeScript)
npx ts-prune

# 미사용 의존성
npx depcheck
```

---

## ✅ 안전한 정리 프로세스

1. 삭제 전 모든 테스트 PASS 확인
2. SAFE 레벨만 삭제
3. 삭제 후 테스트 재실행
4. 테스트 실패 시 롤백

---

## ⚠️ 주의사항

- ✅ 항상 테스트 전후로 실행
- ✅ 작은 배치로 삭제
- ✅ 정리 커밋 분리
- ❌ CAUTION/DANGER 검토 없이 삭제 금지
- ❌ 기능 변경과 섞지 않기
