# /tdd 가이드

> RED-GREEN-REFACTOR 사이클로 안정적인 코드를 작성하는 워크플로우

---

## 📌 TDD 사이클

```
RED → GREEN → REFACTOR → REPEAT

RED:      실패하는 테스트 작성
GREEN:    최소한의 코드로 통과
REFACTOR: 코드 개선 (테스트는 계속 통과)
REPEAT:   다음 기능/시나리오
```

---

## 📌 언제 사용하나요?

| 상황 | 예시 |
|------|------|
| 새 함수/유틸리티 | "날짜 포맷팅 함수" |
| 비즈니스 로직 | "할인 계산 로직" |
| 핵심 기능 | "결제 처리 로직" |
| 리팩토링 | "안전하게 리팩토링" |

---

## 📤 프로세스

### Step 1: 인터페이스 정의
입력/출력 타입 먼저 정의

### Step 2: 실패하는 테스트 작성 (RED)
아직 구현 안 된 코드에 대한 테스트

### Step 3: 최소 구현 (GREEN)
테스트 통과하는 최소한의 코드

### Step 4: 리팩토링 (REFACTOR)
코드 품질 개선 (테스트 계속 통과 유지)

### Step 5: 커버리지 확인
80% 이상 목표

## ✅ Edge Case 체크리스트

| 입력 유형 | 테스트 케이스 |
|----------|--------------|
| 숫자 | 0, -1, MAX_INT, NaN, Infinity |
| 문자열 | "", " ", 매우 긴 문자열, unicode |
