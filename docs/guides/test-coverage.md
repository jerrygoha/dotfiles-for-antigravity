# /test-coverage 가이드

> 테스트 커버리지를 분석하고 80% 달성을 돕는 워크플로우

---

## 📌 언제 사용하나요?

- 커버리지 현황 확인
- 80% 임계값 달성 필요
- CI 커버리지 체크 통과
- 새 기능 후 커버리지 보강

---

## 📤 커버리지 우선순위

| 우선순위 | 커버리지 | 조치 |
|---------|---------|------|
| Critical | < 50% | 즉시 대응 |
| High | 50-70% | 이번 스프린트에 추가 |
| Medium | 70-80% | 곧 개선 |
| OK | > 80% | 유지 |

---

## 📤 모듈별 커버리지 기준

| 모듈 유형 | Line | Branch | 이유 |
|----------|------|--------|------|
| Core/Critical | 90%+ | 85%+ | 비즈니스 핵심 |
| Business Logic | 80%+ | 75%+ | 표준 |
| Utilities | 100% | 100% | 단순, 테스트 가능 |
| UI Components | 70%+ | 60%+ | 테스트 복잡 |

---

## 💡 사용법

```bash
npm test -- --coverage
```

---

## ✅ 베스트 프랙티스

- ✅ Line보다 Branch 커버리지 중시
- ✅ 에러 경로와 edge case 테스트
- ✅ 의미 있는 assertion 작성
- ❌ 숫자만 채우기 위한 테스트 금지

---

## 🔗 권장 흐름

```
/tdd → /test-coverage → /testing (고급 테스트)
```
