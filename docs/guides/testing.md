# /testing 가이드

> 고급 테스팅 기법으로 코드 품질을 높이는 워크플로우

---

## 📌 언제 사용하나요?

| 상황 | 예시 |
|------|------|
| 테스트 품질 향상 | "더 견고한 테스트 필요" |
| Edge case 커버 | "경계값 테스트 추가" |
| Property-based 테스트 | "모든 입력에 대해 검증" |

### `/tdd`와의 차이

| `/tdd` | `/testing` |
|--------|-----------|
| RED-GREEN-REFACTOR | 다양한 테스팅 기법 |
| 새 기능 개발 | 테스트 품질 향상 |

---

## 📤 테스트 구조

```
tests/
├── unit/           # 빠른 격리 테스트 (<100ms)
├── integration/    # 외부 의존성 테스트
├── e2e/            # 전체 사용자 플로우
└── fixtures/       # 테스트 데이터
```

---

## ✅ Edge Case 체크리스트

| 입력 유형 | 테스트 케이스 |
|----------|--------------|
| 숫자 | 0, -1, MAX_INT, NaN, Infinity |
| 문자열 | "", " ", 매우 긴 문자열, unicode, null |
| 배열 | [], 단일 요소, 최대 크기 |

---

## ✅ 커버리지 기준

| 모듈 유형 | Line | Branch |
|----------|------|--------|
| Core/Critical | 90%+ | 85%+ |
| Business Logic | 80%+ | 75%+ |
| Utilities | 100% | 100% |

---

## 🔗 권장 흐름

```
/tdd (기본 테스트) → /testing (고급 테스트) → /test-coverage (커버리지 확인)
```
